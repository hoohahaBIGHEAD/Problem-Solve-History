파일 로그 클래스 만들기
조건
클래스 2개 이상 만들기
파일 관리 클래스
o역할 : 로그 관리 클래스에서 생성된 로그 내용 저장, 날짜 변경시 현재 날짜로 다시 생성
로그 관리 클래스
o역할 : 로그 옵션
           - 파일및 화면 컨트롤 옵션

              (파일 저장만, 화면 출력만, 화면 출력 및 파일 저장)

           - 로그 범위 옵션 (error, waring, event, debug 등)

           로그 생성 포멧

           - yyyymmddhhmmssss:[로그생성 파일][소스 라인] – 로그

<문제 부가 설명>
1. 파일 관리 클래스

    1) 구성에 대한 설명

        - 로그 파일 자동 날짜별 파일 생성(로그는 대부분 일별로 파일이 생깁니다. 그래서 대부분 Thread을 이용하여 날짜를 계속 체크 하다가 날짜가 지나면 그 날짜로 파일을 생성 합니다.)

        - 로그 저장(날짜로 생성된 파일에 여러분들이 출력할 내용을 저장하는것 입니다.)

        - 로그 파일 삭제(로그를 일정 주기을 정하고 그 주기 만큼 보관하고 있다가 삭제을 해 주어야 합니다. 그 이유는 저장 공간이 부족 할 수 있기 때문 입니다.



2. 로그 관리 클래스

    1) 구성 설명

        - 파일 및 화면 컨트롤 옵션(개발시에는 화면을 보편서 로그을 확인 할 수 있지만 프로그램이 설치되면 화면으로 실시간 로그를 볼수 없기 때문에 파일로 저장 해야 합니다. 그래서 옵션에 따라 상항에 맞게 화면 출력을 할 것인지 아니면 파일로 남길것 인지 아니면 둘다 할것인지 정하는것 입니다.)

        - 로그 범위 옵션(debug는 여러분들이 개발시 남기는 로그, error는 프로그램이 갑자기 죽거나 홀딩 될 수 있는 로그, waring은 에러이긴 하나 프로그램이 정상적으로 처리 될수 있는 로그, event는 파일 파일공간이 부족 할때 미리 알려주는 로그라고 생각 하면 됩니다.

        - 로그 생성 포멧(년월일시분초ms까지 남겨야지 여러분이들 충분히 로그을 분석 할 수 있기 때문입니다. 로그 생성파일 이름은 로그을 보았을때 파일 어떤 파일에 몇번째 라인에 서 오류가 발생했는지 알면 바로 소스를 확인 할 수 있기 때문 입니다. 회사 프로젝트를 하다 보면 대부분 로그을 그렇게 남김니다.)



3. 예

    1) 함수 예

         - log_create(file_name, mode, log_type); //mode(파일 저장, 화면 출력, 파일/화면 둘다), log_type(error로그만 출력 할것인지 아니면 warring만 출력 할것인지에 대한 type입니다.)

         - log_print(type, log_data); //type(로그 범위 type(error, waring, event, debug 등), log_data(출력 내용)

    2) 로그 화면 출력및 파일 저장 내용 예

         E: 2021052521203020:socket.c:34line: socket open error

         W: 2021052521203020:file.c:50line: file 저장 공간 부족
